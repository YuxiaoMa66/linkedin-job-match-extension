# License Options

This is a practical summary for choosing a license for this repository.

It is not legal advice, but it should help you make a decision quickly.

## Option 1: MIT

Best when:

- you want the simplest and most common open-source license
- you mainly want to showcase the project publicly
- you do not mind others reusing the code commercially

Pros:

- very short and easy to understand
- widely accepted on GitHub
- ideal for portfolio and demo projects

Tradeoff:

- others can reuse the code with very few restrictions

## Option 2: Apache-2.0

Best when:

- you want a permissive license like MIT
- but you want clearer patent and contribution terms

Pros:

- still business-friendly and permissive
- more explicit legal protection than MIT in some situations
- common for serious engineering projects

Tradeoff:

- a bit longer and more formal than MIT

## Option 3: GPL-3.0

Best when:

- you want strong copyleft
- you want derivative works that distribute modified versions to also stay open source

Pros:

- strongest “stay open” requirement among the common options here

Tradeoff:

- much more restrictive for commercial or closed-source reuse
- may reduce adoption if your goal is portfolio visibility or easy reuse

## My Practical Recommendation

If your main goal is:

- showing the project on GitHub
- letting recruiters, collaborators, and other developers inspect it easily
- avoiding unnecessary friction

Then the most practical choices are:

1. `MIT`
2. `Apache-2.0`

## Short Decision Guide

Choose `MIT` if:

- you want maximum simplicity
- you care more about visibility than restrictions

Choose `Apache-2.0` if:

- you still want permissive reuse
- but prefer a more complete and professional legal framework

Choose `GPL-3.0` if:

- you explicitly want to force distributed derivatives to remain open source

## What I Would Suggest For This Project

For this Chrome extension, I would usually recommend:

- `MIT` for a personal portfolio / public showcase repo
- `Apache-2.0` if you want a slightly more formal and professional default
